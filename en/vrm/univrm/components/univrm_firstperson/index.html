    VRMFirstPerson

    The setting related to first-person display.

    FirstPersonBone

    Normally head bone is assigned as FirstPersonBone. It is the bone that keeps track of the headset in first-person view.

    FirstPersonOffset

    Tracking position is offset from the FirstPersonBone position. We assume the default value [0, 0.06, 0] is the offset between the head and the middle of the eyes.

    Renderers

    The setting for controlling mesh on/off rendered in first-person view.

    For VR applications, you may see the head of the model that is not supposed to be rendered in the scene. To address this problem, we provide the function VRMFirstPerson which renders the model without the head in first-person view. To test out first-person mode, set up render layer in inspector and call first-person setup function. Then, navigate tocamera->cull maskand selectEVERYTHINGbutTHIRDPERSON_ONLY_LAYER.

    The case for head being separated

    Set ThirdPersonOnly on head. Set Both on the rest part.
    Example: for “3D chan” (below), the parts not connected with the head are set to Both. The head is set to ThirdPersonOnly.
    Meshes set to ThirdPersonOnly are not displayed in first-person view

    The case for head not being separated

    Please set Auto. The model except the head portion at runtime is treated as FirstPersonOnly. The original model is treated as ThirdPersonOnly. The part with the weight in the head bone and its descendants will be eliminated.

    The case for no particular setting

    Please set Both.
