    “VRM” means…

    In the past, when handling character and humanoid avatar (3D model) in VR (Virtual Reality), VTuber, etc., it was necessary to develop their own systems for applications and further fine-tune 3D model data due to…

    • The output data made by 3D model creators and modeling tools is different

      • The coordinate system is different, the scale is different, the initial pose is different, the expression is different…
      • Of course, the method importing bones into the program is also different.

    • The format of handling 3D model data is different. Each company has their own specifications, which are more complex than basic settings. Also, the necessary information about their file formats are not fully provided

      • Whether multi-compatible software FBX file is readable by applications, which version of the application can process FBX files correctly…usually users concern about these kinds of issues.

    • There is not enough necessary information available from the viewpoint of using 3D model data as avatar

      • For example, in the first-person view, how to obtain the right position, how to get rid of the head display, which part of the body should be excluded and so on.

    As avatar expression applied in VR grows up very fast, if the situations mentioned above remain unchanged, application developers and 3D model creators will have to spend double or triple effort. To improve current situations, based on humanoid character and avatar, the first step is to:

    • Effectively absorb and unify the (subtle) differences from model data
    • Make handling 3D model easy on application side

    Here we provide the open source software “VRM” featured by platform-independent 3D avatar file format.

    What can you do with VRM？

    The same avatar (3D model data) created by VRM can be used in any VRM-compatible platform. The future will be filled with various kinds of VR applications for sure…

    • Create your own avatar with VRM-compatible character making tool
    • Use the created avatar to host live streaming and then join your friends’ live channels
    • After live streaming, start VR game and explore VR world with your own avatar
    • When getting tired of game, move to the Chat in the VR world. Use the same avatar character to hang out with friends
    • Join the VR study group in VR tomorrow. Of course, the same avatar appearance

    By handling your own avatar (3D model data) with VRM, you can easily go back and forth among various VRM supported applications and games.

    Live broadcasting, video creation, games, chat…. Now the virtual worlds are separated. The first step to connect the separated virtual worlds is VRM.

    VRM features

    VRM can handle humanoid character and avatar. Although the current provided software to read and write VRM is for Unity, VRM itself is platform-independent. Also, it can be handled in other system engines and environments.

    • Available to handle humanoid character (3D model data) without limiting to a specific platform.
    • VRM is based on 3D standard format glTF2.0, which is the format added with constraints and extensions for handling humanoid models. Easy to implement based on this standard format.
    • All files including textures and materials are grouped into one file. You only need to handle one file at one time.
    • The standard composition is decided by scale (1.00 = 1 m), coordinate system (Y-up, -Z direction) and bone structure (conforming to Unity Humanoid, T stance, for each bone no local rotation). Differences resulted by different modeling tools will not affect the correctness of VRM output.
    • Various techniques can be applied to handle character expression such as BlendShape and transparency of material. Differences between techniques can be absorbed and unified through API operation on the VRM side.
    • For the use of avatar in VR, VRM supports the rendering in first-person.
    • A standard implementation of swaying effect that does not rely on physical engine is available for setting character’s hair, clothes and so on.
    • Material (shader) as standard implementation is provided. Not only standard PBR, but also tone shading and Unlit rendering are available for setting.
    • Not only meta information such as title and author name, but also thumbnail and license information specialized for avatars in VR era can be included in VRM file.

    VRM is a 3D avatar format expected to be used among multiple applications in VR era. For example, considering the situation of “VR communication with avatars in the interconnected network”, it is possible to send your own avatar data to other users as they can see your avatar appearance in a shared virtual space.

    In this case, there may be situations that cannot be dealt with by only adopting redistribution rules based on conventional thinking. Therefore, VRM has:

    • Modification / Redistribution rules for model data itself（can be configured from Creative Commons, etc.）

    in addition to

    • The rule for model performable acts

    The new adding above is assumed to be the format in VR era.
